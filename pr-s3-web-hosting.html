Hosting a Static Website on AWS Using EC2

By Samuel Nnanna

This guide walks you through hosting a static website on an AWS EC2 instance. You’ll launch a server, install a web server, upload your site files, and make the site publicly accessible over the web. :contentReference[oaicite:1]{index=1}

1. Choose Your AWS Region

Log in to the AWS Management Console and pick a region close to your users (e.g., us-east-1). :contentReference[oaicite:2]{index=2}

2. Create a Security Group

Security groups control network access to your instance:

• Open AWS Console → VPC → Security Groups  
• Create a security group  
• Add inbound rules:  
  - SSH (port 22) → source: your IP  
  - HTTP (port 80) → source: anywhere (IPv4)

This allows you to connect via SSH and serve web traffic. :contentReference[oaicite:3]{index=3}

3. Launch an EC2 Instance

Create your EC2 virtual server:

  • Go to EC2 → Launch Instance
  • Choose Amazon Linux 2 AMI
  • Select t2.micro (free tier eligible)
  • Create or select a key pair (.pem file)
  • Assign the security group you created
  • Launch and wait for status checks to pass

This instance will host your static website. :contentReference[oaicite:4]{index=4}

4. SSH Into the Instance

Find your instance’s public IPv4 address in the AWS console, then connect from your local terminal (example):

ssh -i "your-key.pem" ec2-user@your-public-ip

Type “yes” if prompted to continue connecting. :contentReference[oaicite:5]{index=5}

5. Install Required Software

Once logged in:

sudo -i  
yum update -y  
yum install httpd -y  
cd /var/www/html

This installs Apache, the web server that will serve your static files. :contentReference[oaicite:6]{index=6}

6. Upload and Prepare Your Website Files

Download or upload your static website files to the EC2 instance (e.g., using wget or scp):

wget https://www.toolplate.com/zip-templates/2135_mini_finance.zip  
unzip 2135_mini_finance.zip  
cp -r 2135_mini_finance/* /var/www/html  
rm -rf 2135_mini_finance 2135_mini_finance.zip

This copies your HTML/CSS/JS assets to Apache’s root directory. :contentReference[oaicite:7]{index=7}

7. Start the Web Server

Enable and start Apache so it serves your website:

systemctl enable httpd  
systemctl start httpd

Your web server is now running. :contentReference[oaicite:8]{index=8}

8. Access Your Static Website

Open your browser and visit the public IPv4 address of your EC2 instance:

http://your-public-ip/pre>

You should now see your static site live on the internet! :contentReference[oaicite:9]{index=9}



9. Clean Up (Optional)


If you no longer need the instance, remember to terminate it in the EC2 dashboard and delete any associated security group and keys to avoid charges. :contentReference[oaicite:10]{index=10}



Wrap-up


This tutorial showed how to deploy a static website using an AWS EC2 instance and the Apache web server. It’s a hands-on way to understand cloud servers, security, and website hosting fundamentals. :contentReference[oaicite:11]{index=11}
